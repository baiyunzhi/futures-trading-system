# 商品期货交易系统 · 系统说明文档

> 标的：中国商品期货（黑色系 / 有色金属 / 农产品）
> 目标：客观、可落地、可验证的短线交易决策辅助系统

---

## 一、系统架构

```
main.py
  ├── data_fetcher.py       数据层：行情获取 + 缓存
  ├── indicators.py         指标层：MA / RSI / MACD / ATR / 布林带 / ADX
  ├── variety_selector.py   选品层：品种综合评分 + 排名
  ├── market_analyzer.py    状态层：市场状态判断（观望 / 轻仓 / 趋势）
  ├── signal_generator.py   信号层：入场 / 出场信号
  ├── risk_manager.py       执行层：仓位计算 + 止损管理
  ├── backtester.py         验证层：历史回测 + 绩效统计
  └── dashboard.py          展示层：Plotly Dash 可视化仪表板
```

## 三、品种评分标准（0-100分）

品种综合评分 = 趋势分(40%) + 动量分(35%) + 波动分(25%)

### 趋势分（0-100）
| 条件                    | 得分       |
|-------------------------|------------|
| MA5 > MA10              | +10        |
| MA10 > MA20             | +15        |
| MA20 > MA60             | +15        |
| ADX 趋势强度（直接映射）| 最多 +40   |

### 动量分（以50为基准，±调整）
| 条件                    | 调整       |
|-------------------------|------------|
| 近5日涨跌幅 × 3         | ±20（上限）|
| 近20日涨跌幅 × 1.5      | ±20（上限）|
| RSI 在 40-65 之间       | +10        |
| RSI < 30 或 > 75（极端）| -10        |

### 波动分（ATR占价格比）
| ATR/价格  | 说明        | 得分     |
|-----------|-------------|----------|
| 0.8%-2.5% | 理想波动    | 80-100   |
| < 0.8%    | 波动过低    | < 60     |
| > 2.5%    | 波动过高    | 递减     |
| 成交量放大（>均量1.2×）| 额外 | +10  |

## 四、市场状态判断

系统将每个品种的市场状态分为 5 级：

| 状态          | 条件                                      | 操作建议            |
|---------------|-------------------------------------------|---------------------|
| ⚪ 观望       | 综合分 < 45 且无布林突破                  | 空仓等待             |
| 🟡 轻仓做多  | 分数 45-64，方向向上，MACD 偏多          | 小仓试多（≤1%风险） |
| 🟡 轻仓做空  | 分数 45-64，方向向下，MACD 偏空          | 小仓试空（≤1%风险） |
| 🟢 趋势做多  | 分数 ≥ 65，ADX>25，MACD金叉，量能放大   | 正常仓位做多（2%风险）|
| 🔴 趋势做空  | 分数 ≥ 65，ADX>25，MACD死叉，量能放大   | 正常仓位做空（2%风险）|

## 五、入场条件（多头示例）

同时满足以下4条才产生入场信号：

1. **方向过滤**：收盘价 > MA20（在均线上方）
2. **触发信号**：MACD 金叉（DIF 上穿 DEA）**或** RSI 从超卖区（<35）回升
3. **量能配合**：当日成交量 > 5日均量 × 1.1
4. **状态允许**：市场状态 ≠ 观望

空头条件对称（价格 < MA20，MACD 死叉 / RSI 超买回落）

## 六、止损与止盈规则

```
止损距离 = 2 × ATR(14)
止盈距离 = 3 × ATR(14)

多头：止损价 = 入场价 - 2×ATR   止盈价 = 入场价 + 3×ATR
空头：止损价 = 入场价 + 2×ATR   止盈价 = 入场价 - 3×ATR
```

**盈亏比 1:1.5**（理论上胜率 >40% 即可盈利）

---

## 七、出场条件（多头）

触发任一条件即平仓：
- 价格跌破止损价（硬止损）
- 价格触及止盈价（锁定利润）
- MACD 死叉 且 价格跌破 MA5（趋势反转信号）

---

## 八、仓位管理

```
单笔风险 = 总资金 × 2%
开仓手数 = 单笔风险 / (2 × ATR × 合约乘数)
最多同时持仓 = 3 个品种
```

---

## 九、回测验证方法

1. 运行 `python main.py` 后系统自动对所有品种执行历史回测
2. 回测区间：2023-01-01 ~ 2024-12-31（可在 `config.py` 中修改）
3. **验证指标**：
   - 胜率 > 40%（目标）
   - 最大回撤 < 15%（红线）
   - 夏普比 > 0.5（目标）
   - 盈亏比（实际）> 1.2

## 十二、参数调优方向

| 参数              | 位置            | 说明                         |
|-------------------|-----------------|------------------------------|
| 品种池            | config.py       | 增减品种                     |
| 评分权重          | config.py       | 调整趋势/动量/波动的权重      |
| 状态阈值（45/65）  | config.py       | 调整激进程度                 |
| ATR 止损倍数(2.0) | config.py       | 越大止损越宽，越小越容易被洗  |
| ATR 止盈倍数(3.0) | config.py       | 控制盈亏比                   |
| 最大同时持仓(3)   | config.py       | 控制分散程度                 |
| 回测区间          | config.py       | 换不同市场环境验证            |

## 十三、风险提示

- 本系统为量化研究工具，不构成投资建议
- 历史回测不代表未来表现
- 商品期货存在较大波动风险，请严格执行止损
- 建议先在模拟账户验证至少 3 个月后再考虑实盘
