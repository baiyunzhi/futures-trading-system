# 四维分析系统初版

减法版输出。

每个品种、每个周期只保留五项：

```text
1. 当前状态
2. 当前观察位
3. 等待原因
4. 准备等级
5. 失效条件
```

运行：

```bash
python 四维分析系统初版/build_site.py
```

输出：

```text
四维分析系统初版/index.html
docs/四维分析系统初版/index.html
```

## 新增品种

新增品种时保持同一套工作流：

```text
1. 在 four_dimension_system/market_system.py 的 SYMBOLS 增加品种
2. 准备该品种最近两个月 daily/hourly CSV
3. 在 四维分析系统初版/build_site.py 的 DATA_FILES 增加数据路径
4. 重新运行 python 四维分析系统初版/build_site.py
```

页面会自动输出该品种的周线、日线、小时线，每个周期包含五项描述和K线图。

## 锚点工作流

每个周期会自动识别锚点状态：

```text
原锚点有效
候选新锚点
锚点待重建
```

规则目的：

```text
1. 不让系统一直盯着已经远离价格的旧最大量K线
2. 后续出现新的放量K线时，先标为候选新锚点
3. 只有价格继续围绕候选新锚点高低点波动，才把它作为新的观察区间
```

## 动态观察位链条

最大成交量K线只作为初始观察区间。

行情后续演变时，系统会识别：

```text
支撑位跌破
支撑转压力
阶段低点被再次跌破
当前观察位下移
```

示例：

```text
PVC日线：5054 -> 4804 -> 4652
```

含义：

```text
5054 原锚点低点失守
4804 阶段支撑失守后转压力
4652 弱势震荡低点失守后成为当前压力观察位
```

当动态观察位链条形成后，系统会同步调整：

```text
当前状态
等待原因
准备等级
失效条件
```

这些项目不再只围绕原最大量K线，而是优先围绕最近有效观察位。

## 选品与准备工作流

页面顶部先按日线四维弱势/强势排序：

```text
价格变化
动态观察位层数
最大量K线后的区间位置
成交量与持仓量态度
```

方向向下时优先观察更弱的品种；方向向上时优先观察更强的品种。

每个周期仍只输出五项，但准备等级会同时合并：

```text
品种优先级
当前观察位距离
弱势震荡是否形成
反抽/回踩区域是否清楚
风险收益比是否达到 1:3
```

如果价格已经远离反抽或回踩区域，即使品种最弱，也只显示等待，不追价。

## 放量事件与量仓确认

系统新增三步观察：

```text
1. 跨周期放量事件队列
   周线、日线、小时线的最大成交量K线统一进入队列。
   后续增加5分钟数据时，5分钟天量也进入同一队列。

2. 最大量K线性质
   根据开盘、收盘、高点、低点、成交量、持仓量变化，描述这根K线是进攻、反击、换手，还是上下影压力/承接。

3. 观察位附近量仓确认
   只有价格回到支撑/压力附近时，才判断成交量和持仓量是否支持突破、反抽失败或回踩有效。
```

规则边界：

```text
远离观察位时，只描述当下量仓变化，不把它当作支撑/压力确认。
接近观察位时，成交量和持仓量才进入准备等级判断。
```
