# 四维分析系统初版

减法版输出。

每个品种、每个周期只保留五项：

```text
1. 当前状态
2. 当前观察位
3. 等待原因
4. 准备等级
5. 失效条件
```

运行：

```bash
python 四维分析系统初版/build_site.py
```

输出：

```text
四维分析系统初版/index.html
docs/四维分析系统初版/index.html
```

## 新增品种

新增品种时保持同一套工作流：

```text
1. 在 four_dimension_system/market_system.py 的 SYMBOLS 增加品种
2. 准备该品种最近两个月 daily/hourly CSV
3. 在 四维分析系统初版/build_site.py 的 DATA_FILES 增加数据路径
4. 重新运行 python 四维分析系统初版/build_site.py
```

页面会自动输出该品种的周线、日线、小时线，每个周期包含五项描述和K线图。

## 锚点工作流

每个周期会自动识别锚点状态：

```text
原锚点有效
候选新锚点
锚点待重建
```

规则目的：

```text
1. 不让系统一直盯着已经远离价格的旧最大量K线
2. 后续出现新的放量K线时，先标为候选新锚点
3. 只有价格继续围绕候选新锚点高低点波动，才把它作为新的观察区间
```
