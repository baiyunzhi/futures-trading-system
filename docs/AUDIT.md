# 交易系统审计报告

更新时间：2026-06-26

## 高风险漏洞

| 问题 | 风险 | 处理结果 |
|---|---|---|
| 信号日收盘价直接成交 | 使用了当天收盘后才知道的信息，回测收益偏乐观 | 已改为下一根 K 线开盘价成交 |
| 只用收盘价判断止损止盈 | 日内触发硬止损会被漏掉 | 已改为用 high/low 检查日内触发 |
| 固定 1 手开仓 | ATR 风险管理没有真正生效 | 已按 ATR、资金风险、合约乘数计算手数 |
| 交易净 PnL 漏算开仓手续费 | 绩效统计偏高 | 已计入开平双边手续费 |
| 未平仓交易不进入绩效 | 期末风险被隐藏 | 已在最后一根 K 线强制平仓 |
| 实际成交价与信号价不一致 | 隔夜跳空后止损/止盈距离失真 | 已按次日开盘真实成交价重算风控价格 |
| 行情 OHLCV 未做完整校验 | 缺失、非正数、重复日期会污染回测 | 已加入数据质量过滤 |

## 中风险漏洞

| 问题 | 风险 | 处理结果 |
|---|---|---|
| 缺少滑点模型 | 成交价格过于理想 | 已按品种 tick size 加入滑点 |
| 净值曲线只记录现金 | 持仓期间回撤失真 | 已加入浮盈浮亏盯市 |
| 静态导出依赖 Dash | GitHub Pages 生成可能失败 | 已改为静态导出不强依赖 Dash |
| 数据失败自动仿真 | 用户可能误把演示结果当真实行情 | 页面已标记仿真数据提示 |
| 仿真数据不可复现 | Python hash 随进程变化，回测样本不稳定 | 已改为稳定哈希种子 |
| 模拟盘日内熔断不强平 | 停止新信号但已有仓位仍暴露 | 已触发强平并停止接收新信号 |

## 2026-06-27 核心改造落地

| 目标 | 落地结果 |
|---|---|
| 用高低点结构替代 MA 趋势判断 | 新增 `trade_decision.py`，基于结构高低点、突破、密度和区间宽度分型 |
| 行情分成趋势、震荡、突破、缠绕 | `classify_regime()` 输出 TREND / RANGE / BREAKOUT / CHOP |
| 每种行情只匹配一种策略 | TREND→趋势跟随，BREAKOUT→突破回踩，RANGE→区间反转，CHOP→禁止交易 |
| 每笔交易先算风险再算手数 | `calc_risk_first_lots()` 按止损距离、合约乘数和账户风险计算手数 |
| 回测改成统一账户组合回测 | 新增 `portfolio_backtester.py`，统一资金、组合持仓、板块限制和组合净值曲线 |
| 滚动样本外动态准入 | 新增 `portfolio_filter.py`，每个交易日只用此前 365 天已平仓交易决定品种是否准入 |
| 准入结果可视化 | 静态页面的“可执行交易决策”会显示滚动禁入原因，并在统一账户回测中同步执行 |
| 移除均线/MACD决策依赖 | 主交易链路切换到 `strategy_structure.py`，信号只使用滚动高低点结构和 ATR 风控 |
| 页面去指标化 | K线图移除均线和 MACD 子图，行情描述移除 MA/MACD/RSI/ADX 指标段 |
