# 四维客观行情交易系统

## 核心目标

系统先描述行情，再决定是否下注。

判断顺序：

1. 价格：结构、突破、回踩、区间、关键高低点
2. 成交量：放量、缩量、量价配合
3. 持仓量：增仓、减仓、资金持续性
4. 时间：周线、日线、小时线所处阶段

最终输出：

- 当下行情正在干什么
- 不确定性来自哪里
- 什么时候下注
- 怎样下注
- 下注后哪里失效

## 多周期职责

| 周期 | 作用 | 输出 |
|---|---|---|
| 周线 | 大环境过滤 | 多、空、中性 |
| 日线 | 主状态判断 | 趋势、突破、震荡、蓄势 |
| 小时线 | 日内触发 | 突破、回踩、假突破 |

当前数据源主要是日线。系统会自动给出周线和日线判断；如果数据没有小时级时间戳，小时线显示为不可用，不强行伪造日内信号。

## 行情状态

| 状态 | 客观含义 | 动作 |
|---|---|---|
| 价格向上突破关键高点 | 多头尝试离开原区间 | 等回踩或站稳后做多 |
| 价格向下跌破关键低点 | 空头尝试离开原区间 | 等反抽或收稳后做空 |
| 高低点抬升 | 趋势上行 | 只找多头试错 |
| 高低点下移 | 趋势下行 | 只找空头试错 |
| 震荡平衡区 | 无明确方向 | 不下注，等突破 |
| 收敛蓄势期 | 波动收缩，等待方向 | 只准备计划 |
| 扩张突破期 | 波动扩大，方向选择 | 可执行触发交易 |

## 量价仓持续性

| 价格 | 成交量 | 持仓量 | 判断 |
|---|---|---|---|
| 上涨 | 放大 | 增加 | 多头新增，持续性强 |
| 上涨 | 放大 | 减少 | 空头平仓，持续性降级 |
| 下跌 | 放大 | 增加 | 空头新增，持续性强 |
| 下跌 | 放大 | 减少 | 多头离场，急跌后可能衰竭 |
| 横盘 | 缩量 | 增加 | 蓄势，等待突破 |
| 横盘 | 缩量 | 持平 | 平衡，无交易价值 |

没有持仓量数据时，系统不会删除信号，但会自动降低持续性评分。

## 下注规则

### A级机会

必须满足：

1. 日线方向明确
2. 周线不冲突
3. 价格处于趋势运行期或扩张突破期
4. 成交量支持方向
5. 持仓量支持方向，或无持仓量但价格和成交量足够强
6. 已经计算出明确止损和手数

执行：

```text
风险比例：1.2%
入场：突破站稳或回踩不破后转强
止损：结构关键位外侧 + 0.5 ATR
目标：1R 减仓，2R 或结构移动止盈
```

### B级机会

满足方向初步一致，但持续性不足。

执行：

```text
风险比例：0.6%
只允许试错
触发失败立即退出
不加仓
```

### 观察

出现以下情况不下注：

- 周线和日线冲突
- 日线处于震荡平衡区
- 成交量和持仓量不支持方向
- 缺少足够K线
- 止损距离过大导致无法按风险开仓

## 日内首小时方案

启用条件：

```text
数据源必须有分钟或小时K线
必须有成交量
最好有持仓量
```

做多触发：

```text
首小时放量形成区间
价格突破首小时高点
突破后收盘站稳
回踩不跌回区间
再次放量转强
持仓量增加或不明显下降
```

做空触发：

```text
首小时放量形成区间
价格跌破首小时低点
跌破后收盘站稳
反抽不回到区间
再次放量转弱
持仓量增加或不明显下降
```

假突破：

```text
突破后2根15分钟K线内重新收回首小时区间
= 假突破
= 同方向禁止再次开仓
```

## 持仓量数据落地方案

### 1. 总持仓量

首选 akshare 日线行情：

```text
akshare.futures_zh_daily_sina(symbol="RB0")
```

该接口返回字段：

```text
date / open / high / low / close / volume / hold / settle
```

系统已将：

```text
hold -> open_interest
```

总持仓量用于判断行情持续性：

```text
价格上涨 + 持仓量增加 = 多头新增，持续性强
价格上涨 + 持仓量减少 = 空头平仓，持续性降级
价格下跌 + 持仓量增加 = 空头新增，持续性强
价格下跌 + 持仓量减少 = 多头离场，持续性降级
```

### 2. 会员持仓排名

补充 akshare 新浪持仓排名：

```python
from open_interest_sources import fetch_sina_member_position_rank

rank = fetch_sina_member_position_rank("RB2410", "2024-06-21")
```

返回：

```text
成交量
多单持仓
空单持仓
```

注意：

```text
RB0 / I0 / CU0 是主力连续代码，适合日线行情。
RB2410 / I2409 / CU2408 是真实合约代码，适合查会员持仓排名。
```

### 3. 交易所校验

实盘前必须用交易所官网数据交叉校验：

```text
上期所 SHFE
大商所 DCE
郑商所 CZCE
广期所 GFEX
能源中心 INE
```

校验重点：

- 持仓量是否缺失
- 换月时主力合约是否切换
- 持仓量是否与交易所公布值一致
- 数据是否有延迟
- 会员排名是否对应真实合约而非主力连续
