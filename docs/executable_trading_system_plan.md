# 可执行交易系统改造计划

目标：拿到任何一个期货品种，都能客观判断能不能做、做多还是做空、哪里进、哪里止损、仓位多少、什么情况下退出。

## 一、系统总目标

把当前系统升级为：

1. 品种识别系统
2. 行情结构判断系统
3. 策略选择系统
4. 入场执行系统
5. 风控仓位系统
6. 出场管理系统
7. 组合回测验证系统
8. 可视化决策面板

## 二、核心原则

1. 不靠主观感觉。
2. 不靠单一指标。
3. 不预测，只描述当下结构。
4. 先判断能不能交易，再判断方向。
5. 先确定亏多少钱，再决定下多少手。
6. 任何交易必须有入场、止损、退出、失效条件。
7. 所有规则必须能回测、能复盘、能复现。

## 三、第一阶段：重构行情结构判断

目标：拿到任意品种，先判断行情属于哪一种。

需要识别：

1. 上升趋势：高点不断创新高，低点不断抬高。
2. 下降趋势：低点不断创新低，高点不断下移。
3. 宽幅震荡：高低点无明显方向，但区间边界清楚。
4. 窄幅缠绕：K线密集、方向混乱，禁止交易。
5. 突破行情：突破关键高点/低点，并伴随波动或成交量放大。
6. 假突破：突破后快速跌回区间，反向风险加大。

## 四、第二阶段：建立交易分型

系统不再用一套策略打所有行情，而是分三类。

### 1. 趋势跟随策略

适用：

- 上升趋势做多
- 下降趋势做空
- 不做震荡和缠绕

入场：

- 趋势成立
- 回踩不破前低/前高
- 再次突破小级别高点/低点

止损：

- 多头放在最近结构低点下方
- 空头放在最近结构高点上方
- 加 ATR 缓冲

出场：

- 结构破坏
- 移动止损触发
- 出现反向结构

### 2. 区间震荡策略

适用：

- 上下边界清晰
- 没有明显趋势
- 波动足够

入场：

- 接近下沿做多
- 接近上沿做空
- 必须有止损空间和盈亏比

禁止：

- 区间太窄
- K线过密
- 突破后追反向

### 3. 突破策略

适用：

- 长时间压缩
- 波动收敛
- 突破关键高低点

入场：

- 突破后不立刻追
- 等回踩确认
- 回踩不跌回区间再入场

止损：

- 放回区间内部
- 或放在突破K线反向端

## 五、统一交易决策模板

每个品种最终必须输出：

1. 是否可交易：是 / 否
2. 当前结构：上升 / 下降 / 震荡 / 缠绕 / 突破 / 假突破
3. 交易方向：多 / 空 / 观望
4. 策略类型：趋势 / 区间 / 突破 / 禁止
5. 参考入场价
6. 止损价
7. 第一目标位
8. 移动止损规则
9. 单笔风险金额
10. 建议手数
11. 失效条件
12. 当前交易评分
13. 是否允许实盘模拟

## 六、风控系统

必须从“信号驱动”改成“风险驱动”。

规则：

1. 单笔最大亏损：账户资金 0.5%-2%
2. 同时最多持仓：3 个
3. 同板块最多持仓：1 个
4. 连续亏损 3 笔：仓位减半
5. 当日亏损超过 3%：停止开仓
6. 单周亏损超过 6%：停止交易
7. 止损距离太远：放弃交易
8. 盈亏比小于 1:2：放弃交易

## 七、真正组合回测

必须支持：

1. 多品种共用一份资金
2. 同时持仓限制
3. 板块相关性限制
4. 真实手续费
5. 滑点
6. 保证金占用
7. 涨跌停无法成交
8. 主力合约换月
9. 夜盘缺口
10. 强制平仓逻辑

## 八、样本外验证

验证方式：

1. 训练期：2020-2022
2. 验证期：2023
3. 样本外：2024-2026
4. 滚动窗口回测
5. 参数扰动测试

合格标准：

1. 不是只靠一个品种赚钱
2. 不是只靠某一年赚钱
3. 参数微调后仍然有效
4. 最大回撤可控
5. 样本外不崩溃
6. 连续亏损后系统还能恢复

## 九、可视化改造

页面要从“展示指标”改成“交易决策面板”。

新增面板：

1. 当前可交易品种
2. 禁止交易品种
3. 品种结构分类
4. 策略类型分布
5. 当前候选交易
6. 入场/止损/目标位图示
7. 仓位建议
8. 组合风险暴露
9. 板块风险暴露
10. 回测可信度评分
11. 样本外验证结果
12. 交易复盘表

## 十、优先落地的五件事

1. 用高低点结构替代 MA 趋势判断。
2. 把行情分成趋势、震荡、突破、缠绕。
3. 每种行情只匹配一种策略。
4. 每笔交易必须先算风险，再算手数。
5. 回测改成统一账户组合回测。

